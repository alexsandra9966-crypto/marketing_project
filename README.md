Прогнозирование вероятности покупки клиента в течение 90 дней
Описание проекта
В рамках проекта была решена задача машинного обучения в области маркетинговой аналитики.
Цель проекта — построить модель, способную предсказать вероятность совершения клиентом покупки в течение следующих 90 дней на основе истории покупок и взаимодействия с маркетинговыми рассылками.
В ходе работы были выполнены:
* предобработка данных;
* исследовательский анализ данных (EDA);
* разработка признаков (Feature Engineering);
* обучение и сравнение моделей;
* оптимизация качества модели;
* финальное тестирование.
Целевой признак:
* 0 — клиент не совершит покупку в целевом периоде;
* 1 — клиент совершит покупку в течение следующих 90 дней.
Основная метрика качества — ROC-AUC.

Описание данных
В проекте использованы три набора данных.
apparel-purchases
История покупок клиентов.
Описание полей:
* client_id — идентификатор клиента;
* quantity — количество товаров;
* price — стоимость товара;
* category_ids — вложенные категории товара;
* date — дата покупки;
* message_id — идентификатор сообщения.
apparel-messages
История маркетинговых рассылок.
Описание полей:
* bulk_campaign_id — идентификатор кампании;
* client_id — идентификатор клиента;
* message_id — идентификатор сообщения;
* event — тип события;
* channel — канал коммуникации;
* date — дата события;
* created_at — дата и время создания сообщения.
apparel-target_binary
Целевой признак.
Описание полей:
* client_id — идентификатор клиента;
* target — факт совершения покупки.

Этапы работы
1. Предобработка данных
* проверка типов данных;
* обработка пропусков;
* удаление дубликатов;
* преобразование форматов дат.
2. Исследовательский анализ данных (EDA)
* анализ целевого признака;
* исследование поведения клиентов;
* анализ рассылок и покупок.
3. Разработка признаков (Feature Engineering)
Созданы:
* клиентская витрина;
* числовые признаки;
* признаки покупательской активности;
* признаки категорий товаров;
* признаки каналов коммуникации;
* признаки событий.
4. Машинное обучение
Обучены модели:
* Logistic Regression;
* Random Forest.
Для улучшения качества использованы:
* масштабирование признаков;
* подбор гиперпараметров (GridSearchCV);
* отбор наиболее информативных признаков.

Результаты
Полученные метрики:
Модель	ROC-AUC
Logistic Regression (baseline)	0.7515
Random Forest	0.7315
Logistic Regression (оптимизированная)	0.7606
Финальная модель (тест)	0.7750
Дополнительная метрика:
* F1-score = 0.0676
Наиболее важные признаки:
* время с последней покупки;
* средний чек;
* суммарная выручка;
* количество покупок;
* конверсия сообщений в покупки;
* активность клиента.
